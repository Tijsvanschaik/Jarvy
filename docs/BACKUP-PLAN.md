# Backup plan

Prepare a locally playable backup video of the five routines on the presentation machine and a second offline copy. Do not embed participant data, API keys, or live transcripts.

Graceful degradation:

- Realtime unavailable: narrate the flow and use the offline rehearsal/report.
- Transcription delayed: continue only with consent; queued WAV jobs persist for later processing.
- Exa unavailable: use the validated local signal library; do not invent current facts.
- Camera denied/busy: skip routine 4 or describe the intended explicit-consent path; never request broader OS permissions.
- Image generation unavailable: keep text artifacts and text-first recap slides.
- Recap provider unavailable: use stored transcript, summaries, and notes; do not create unsupported participant claims.
- Shortcut unavailable: use visible power/operator controls.
- Network lost: hard close live providers, preserve the queue, and play the backup video.

The operator should announce degraded features plainly. Never represent fixture output or provisional routine 3 as a live municipal system.
